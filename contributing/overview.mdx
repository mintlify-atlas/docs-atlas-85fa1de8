Contributions are welcome! Whether you are fixing a bug, adding a feature, or improving a translation, this guide will get you set up.

## Prerequisites

- [Node.js](https://nodejs.org/) 18 or later
- [pnpm](https://pnpm.io/) package manager
- [Expo Go](https://expo.dev/go) on a physical device, **or** an Android emulator / iOS simulator
- For Android builds: Android Studio with an emulator configured
- For iOS builds: Xcode on macOS

## Getting started

<Steps>
  <Step title="Fork and clone the repository">
    Fork the [VShopApp/mobile](https://github.com/VShopApp/mobile) repository on GitHub, then clone your fork:

    ```bash
    git clone https://github.com/<your-username>/mobile.git vshop
    cd vshop
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    pnpm install
    ```
  </Step>

  <Step title="Create your .env file">
    Copy the example environment file and fill in at minimum `EXPO_PUBLIC_API_URL`:

    ```bash
    cp .env.example .env
    ```

    See the [Environment Variables](/configuration/environment) page for a description of every variable.
  </Step>

  <Step title="Start the development server">
    <CodeGroup>
      ```bash Start (Expo Go)
      pnpm start
      ```

      ```bash Android
      pnpm android
      ```

      ```bash iOS
      pnpm ios
      ```
    </CodeGroup>

    Scan the QR code with Expo Go, or wait for the emulator to launch automatically.
  </Step>
</Steps>

## Code quality

Run the linter before opening a pull request:

```bash
pnpm lint
```

ESLint is configured via `.eslintrc.js` extending the `expo` preset, which covers React, React Native, and TypeScript rules. TypeScript strict mode is enabled in `tsconfig.json` — all type errors must be resolved before a build will succeed.

## Project structure

| Directory | Contents |
|-----------|----------|
| `app/` | File-based routing screens powered by [Expo Router](https://expo.github.io/router/). Each file maps directly to a route. |
| `components/` | Reusable React Native components shared across screens. |
| `hooks/` | [Zustand](https://zustand-demo.pmnd.rs/) stores and custom React hooks that manage global and local state. |
| `utils/` | API call helpers, localization setup, analytics, and other utilities. |
| `assets/` | Images, fonts, and `i18n/` JSON translation files. |

<Note>
  The `~` path alias (configured in `tsconfig.json`) maps to the repo root. Use `~/components/Foo` instead of relative paths like `../../components/Foo`.
</Note>

## Contributing translations

VShop translations are community-contributed via Weblate. No code changes are needed to add or improve a translation. See the [Translations](/contributing/translations) page for step-by-step instructions.
