---
title: "Environment Variables"
description: "Configure VShop by setting environment variables before building or running the app."
---

VShop uses [Expo's built-in environment variable support](https://docs.expo.dev/guides/environment-variables/). Variables prefixed with `EXPO_PUBLIC_` are bundled into the client at build time and are accessible anywhere in the JavaScript code via `process.env`.

<Note>
  Never store secrets in `EXPO_PUBLIC_` variables — they are embedded in the app bundle and visible to anyone who inspects it. Use them only for non-sensitive configuration such as public API URLs and analytics domains.
</Note>

## Setup

Copy `.env.example` to `.env` at the repo root and fill in the values before running or building the app:

```bash
cp .env.example .env
```

```bash .env
EXPO_PUBLIC_API_URL=https://api.vshop.one

# Optional
EXPO_PUBLIC_STRIPE_PUBLIC_KEY=pk_live_xxxxxxxxxxxxxxxxxxxx
EXPO_PUBLIC_PLAUSIBLE_URL=https://plausible.io
EXPO_PUBLIC_PLAUSIBLE_DOMAIN=vshop.one
```

## Variables

<ParamField path="EXPO_PUBLIC_API_URL" type="string" required>
  Base URL for the VShop backend API. All API requests are sent to this origin. Must include the scheme and host with no trailing slash.

  **Example:** `https://api.vshop.one`
</ParamField>

<ParamField path="EXPO_PUBLIC_STRIPE_PUBLIC_KEY" type="string">
  Stripe publishable key used to process optional in-app donations. When omitted, the donation flow is disabled and no Stripe SDK calls are made.

  **Example:** `pk_live_xxxxxxxxxxxxxxxxxxxx`
</ParamField>

<ParamField path="EXPO_PUBLIC_PLAUSIBLE_URL" type="string">
  Base URL of the Plausible Analytics server that receives event data. Must be set together with `EXPO_PUBLIC_PLAUSIBLE_DOMAIN`; if either is missing, analytics are silently disabled.

  **Example:** `https://plausible.io`
</ParamField>

<ParamField path="EXPO_PUBLIC_PLAUSIBLE_DOMAIN" type="string">
  The site domain registered in your Plausible dashboard. Events are tagged with this value so they appear under the correct site.

  **Example:** `vshop.one`
</ParamField>

## How Expo exposes variables

Expo replaces every reference to `process.env.EXPO_PUBLIC_*` with its literal string value at bundle time using Babel. This means:

- Variables are **read-only** at runtime — you cannot change them without rebuilding.
- Only variables that start with `EXPO_PUBLIC_` are inlined. Any other variable in `.env` is ignored by the Expo bundler (though it may still be read by Node.js scripts such as EAS build hooks).
- The `.env` file should be listed in `.gitignore` to avoid committing real credentials.
